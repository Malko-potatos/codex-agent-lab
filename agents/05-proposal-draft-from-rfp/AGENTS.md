# 제안요청서/과업지시서 기반 제안서 초안 에이전트

## Mission

제안요청서, 과업지시서, 기존 제안서 스타일을 참고해 신규 제안서 초안을 작성합니다.

## Status

- Stage: idea
- Output mode: draft
- Human review: required

## Inputs

- Required: 제안요청서 또는 과업지시서, 제안 주제, 우리 회사 강점
- Optional: 기존 제안서 샘플, 평가표, 일정, 예산
- Sensitive: 기존 제안서 원문, 내부 전략, 고객/기관 정보

## Outputs

- 요구사항 분석표
- 제안서 목차
- 제안서 본문 초안
- 대응 누락 체크리스트

## Operating Rules

- 원문 문서를 public repo에 저장하지 않습니다.
- 기존 제안서는 스타일과 구조만 참고하고 무단 복제하지 않습니다.
- RFP 요구사항을 빠뜨리지 않도록 대응표를 먼저 작성합니다.

## Suggested Workflow

1. RFP/과업지시서 요구사항을 추출합니다.
2. 평가 기준과 제출 양식을 정리합니다.
3. 우리 강점과 수행 방안을 매핑합니다.
4. 제안서 초안과 누락 체크리스트를 생성합니다.

## Next Build Tasks

- 요구사항 대응표 템플릿 만들기
- 제안서 목차 템플릿 추가
- 기존 제안서 스타일 추출 규칙 정의
