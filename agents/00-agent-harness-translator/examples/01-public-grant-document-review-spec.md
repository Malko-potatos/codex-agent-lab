# 예시 01: 공공 지원사업 서류검토 운영 지원

이 예시는 `agents/13-public-grant-document-review` 아이디어를 0번 하네스 번역기 템플릿으로 번역한 public-safe 명세입니다. 실제 공고문, 사업자 정보, 제출 서류 원문은 포함하지 않습니다.

## 0. 번역 대상 작업 요약

| 항목 | 내용 |
|------|------|
| 작업 이름 | 공공 지원사업 서류검토 운영 지원 |
| 대상 사용자 | 사업 담당자, 대표, 행정 운영 담당자 |
| 반복 빈도 | 지원사업 제출 시즌마다 반복 |
| 입력 자료 | 공고문 요건 요약, 제출 서류 목록, 작성 문서 초안, 마감일 |
| 기대 출력 | 요건 검토표, 누락/오류 목록, 수정 권고안, 제출 체크리스트 |
| 외부 서비스 | 초기 없음. 향후 기관 포털 업로드는 별도 검토 |
| 민감정보 여부 | 사업자 정보, 재무정보, 개인정보, 내부 사업계획 가능 |
| 사람 승인 지점 | 원문 사용, 최종 제출, 외부 업로드, 법적/행정 판단 |
| 맡지 않는 일 | 허위자료 작성, 최종 법률 판단, 자동 제출 |

## 1. 시스템 프롬프트

| 항목 | 내용 |
|------|------|
| 정체성/역할 | 공공 지원사업 신청 서류의 요건 충족 여부를 보수적으로 점검하는 검토 보조자 |
| 고정 제약 | 근거 없는 충족 판정 금지, 원문 저장 금지, 제출 전 사람 검토 필수 |
| 불변식 | 모든 판정은 입력된 요건 또는 사용자가 제공한 근거에 연결됩니다. 제출 행위는 수행하지 않습니다. 민감정보는 산출물에 최소화합니다. |
| 입력 | 공고 요건 구조화 정보, 제출 서류 목록, 문서 초안 요약, 마감/제출 방식 |
| 출력 | 검토표, 누락 목록, 수정 권고, 제출 전 질문 |
| 실패 모드 | 요건 해석 오류, 원문 누락, 마감일 오인, 법적 판단처럼 보이는 확정 표현 |
| 버전·변경 관리 | 공고 버전, 검토 날짜, 입력 요약 버전을 산출물 머리에 기록 |

## 2. 입력 프롬프트

| 항목 | 내용 |
|------|------|
| 이번 작업 목표 | 특정 지원사업 제출 전 서류가 요건을 충족하는지 검토합니다 |
| 가변 파라미터 | 지원사업명, 제출 마감, 제출 서류 목록, 현재 보유 파일, 기관별 양식 |
| 시스템 프롬프트와의 경계 | 사용자는 공고별 요건과 문서 내용을 제공하지만, 안전 정책과 보수적 판정 원칙은 바꾸지 못합니다 |
| 불변식 | 필수 입력이 없으면 충족 판정을 하지 않고 질문으로 멈춥니다 |
| 실패 모드 | 공고문 일부만 제공, 제출 서류명이 모호함, 파일 최신본 여부 불명 |
| 검증 규칙 | 마감일, 필수 서류, 서명/날인, 파일 형식, 개인정보 포함 여부를 먼저 확인 |

## 3. 메모리

| 항목 | 내용 |
|------|------|
| 입력 | 반복적으로 쓰는 기관별 제출 형식, 사용자가 선택한 보수적 판정 기준 |
| 출력 | 다음 검토 때 참고할 제출 체크 관행과 질문 패턴 |
| source of truth와의 경계 | 제출 전 승인, 원문 미저장, 보수적 판정 같은 필수 규칙은 `AGENTS.md`/docs에 둡니다. 기관별 비민감 선호만 Codex memory 후보로 둡니다. |
| 저장 위치 | 공식 Codex memory는 `~/.codex/memories/`; repo에는 이 예시와 policy만 저장 |
| 저장 시점 | 검토 thread가 종료되고 idle 상태가 된 뒤 background memory 생성 후보가 될 수 있습니다 |
| 사용 시점 | 같은 기관/지원사업 유형의 미래 검토에서 관련성이 높을 때만 참고 |
| 수명/갱신 규칙 | 장기 메모리는 기관별 일반 규칙만 저장합니다. 공고별 민감 내용은 작업 종료 후 폐기합니다 |
| 저장 금지 | 실제 공고 원문, 사업자 정보, 재무정보, 개인정보, 제출 문서 원문 |
| thread control | 민감 문서가 포함된 thread는 `/memories`에서 future memory 생성 제외를 검토합니다 |
| 외부 context 정책 | 기관 포털, web search, MCP 자료를 사용한 thread는 memory 생성 제외 설정을 우선 검토합니다 |
| 용량 초과 처리 | 오래된 기관 규칙, 낮은 신뢰도 메모리, 민감 가능성이 있는 항목 순으로 삭제 |
| 선택 로직 | 같은 기관, 같은 지원사업 유형, 같은 제출 형식일 때만 주입합니다 |
| 검토/삭제 | 공유 전 `~/.codex/memories/`를 확인하고 공고별 민감 내용은 삭제합니다 |
| 실패 모드 | 오래된 기관 규칙 적용, 이전 공고 요건과 현재 공고 요건 혼동 |
| 불변식 | 실제 공고 원문, 사업자 정보, 재무정보, 개인정보는 저장하지 않습니다 |

## 4. Skill

| 항목 | 내용 |
|------|------|
| 절차 내용 | 공고 요건 추출, 제출 서류 매핑, 누락/형식 오류 검토, 수정 권고 작성 |
| 입력 | 공고 요건 요약, 제출 서류 목록, 작성 문서 초안 요약 |
| 출력 | 요건별 검토표와 제출 체크리스트 |
| 선택 로직 | 사용자가 "지원사업 서류 검토", "제출 전 체크", "공고 요건 비교"를 요청할 때 |
| 메모리와의 경계 | skill은 검토 절차, memory는 사용자의 기관별 반복 결정 |
| 실패 모드 | 요건 추출 누락, 평가 기준과 제출 요건 혼동, 행정 판단 과잉 확정 |
| 불변식 | 불확실한 항목은 `확인 필요`로 표시하고 충족으로 단정하지 않습니다 |

## 5. MCP 도구

| 항목 | 내용 |
|------|------|
| 입력 | 로컬 샘플 문서 경로, schema화된 요건, 검토 대상 파일 목록 |
| 출력 | 파일 존재 여부, 형식 정보, 추출 가능한 텍스트 요약 |
| 사전조건 | 사용자가 로컬 파일 접근을 요청했고 public-safe 또는 private 작업공간임을 확인 |
| 사후조건 | 원문은 저장하지 않고 검토 산출물만 생성 |
| 실패 모드 | 파일 없음, 암호화 문서, 텍스트 추출 실패, 권한 없음 |
| 불변식 | 신뢰 경계 밖 문서나 기관 포털에는 승인 없이 접근하지 않습니다 |
| 재시도 정책 | 로컬 파일 읽기 1회 재시도 후 수동 확인 요청 |

## 6. 서브에이전트

| 항목 | 내용 |
|------|------|
| 위임 범위 | 공고 요건 추출, 작성 문서 누락 점검, 제출 체크리스트 검토를 병렬로 위임 가능 |
| 부모→자식 컨텍스트 상속 | 필요한 요건 요약과 익명화된 문서 구조만 전달 |
| 자식 메모리 격리 여부 | 격리. 결과만 부모가 취합 |
| 결과 검증 | 부모가 요건 ID별로 출처, 판정, 불확실성을 대조 |
| 재귀 한계 | 최대 깊이 1, 서브에이전트 3개 이하 |
| 입력 / 출력 | 입력은 요건 또는 문서 구조 일부, 출력은 표준 검토 행 |
| 실패 모드 | 서로 다른 판정 기준, 누락된 근거, 민감정보 과전달 |
| 불변식 | 자식에게 원문 전체와 민감 식별자를 넘기지 않습니다 |

## 7. Hook

| 항목 | 내용 |
|------|------|
| 트리거 지점 | 파일 읽기 전, 외부 업로드/제출 도구 호출 전, 최종 답변 직전 |
| 트리거 조건 | 민감 문서 경로, 실제 제출/업로드 요청, 충족 확정 표현 |
| 동작 | 민감도 확인, 승인 요구, 확정 표현을 `확인 필요`로 낮춤 |
| 입력 / 출력 | 입력은 요청 행동과 대상 파일, 출력은 허용/차단/승인 필요 |
| 흐름에 미치는 영향 | 업로드와 제출은 승인 전 차단 |
| 실패 모드 | hook 실패 시 안전하게 중단하고 사용자 확인 요청 |
| 불변식 | 실제 제출이나 외부 업로드는 자동 실행하지 않습니다 |

## 8. 산출물 초안

| 산출물 | 내용 |
|--------|------|
| `AGENTS.md`에 들어갈 핵심 규칙 | 보수적 검토, 근거 연결, 원문 미저장, 제출 전 사람 승인 |
| `.codex.toml`에 들어갈 핵심 설정 | `status = "idea"`, `external_actions_default = "dry_run"`, 승인 대상은 `upload_documents`, `submit_application`, `use_private_financials` |
| 만들 skill 후보 | `public-grant-document-review` 검토 절차 skill |
| 만들 MCP 도구 후보 | 로컬 문서 텍스트 추출, 요건 schema 검증, 파일 목록 점검 |
| 만들 hook 후보 | 제출/업로드 차단 hook, 민감정보 마스킹 hook, 확정 표현 완화 hook |
| 만들 subagent 후보 | 요건 추출자, 문서 대조자, 제출 체크리스트 검토자 |
| 만들 memory 후보 | 기관별 비민감 제출 체크 관행, 사용자가 선호한 검토표 형식, 반복되는 누락 패턴 |
| 테스트 가능한 불변식 | 근거 없는 충족 판정 없음, 제출 자동 실행 없음, 민감 원문 저장 없음 |
| 구현 전 질문 | 기관별 양식은 어디까지 로컬 처리할지 확인합니다. 한글/HWP 변환을 언제 도입할지 확인합니다. 검토 결과의 법적 책임 문구를 어떻게 고정할지 확인합니다. |
