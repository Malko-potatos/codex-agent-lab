사용자 요청

지원사업 서류검토 에이전트를 만들어 주세요. 제출서류 폴더와 공고문을 보고 누락 서류, 자격조건 불충족, 파일명 불일치를 점검해 주세요. 결과는 통과 / 보류 / 결격 분류표로 만들면 좋겠습니다. 가능하면 e나라도움 제출 직전 정리까지 해 주세요.

요청 해석

이 요청은 공고문과 제출서류 폴더를 비교해서, 사람이 제출 전에 확인할 수 있는 서류검토 표를 만들고 싶다는 뜻으로 보입니다.

사용자는 단순히 파일 목록을 정리하려는 것이 아니라, 제출 전에 놓친 서류가 있는지, 파일명이 공고 기준과 맞는지, 자격조건에서 위험한 항목이 있는지를 한 번에 보고 싶어 합니다. 그래서 이 에이전트의 목적은 자동 제출보다 “사람이 마지막으로 확인할 수 있는 검토표 만들기”에 더 가깝습니다.

하지만 지금 요청만으로 바로 에이전트를 만들면 목적대로 작동하지 않을 수 있습니다. 제출서류 폴더가 실제 고객 문서인지 샘플인지, 에이전트가 폴더를 직접 읽어도 되는지, 자격조건을 판단할 신청자 정보가 어디에 있는지, `결격`을 최종 판정처럼 써도 되는지 아직 정해지지 않았기 때문입니다.

예를 들어 공고문에는 지방세 완납이 필요하다고 적혀 있어도, 파일명만 보고 실제 완납 여부를 확인할 수는 없습니다. 이 상태에서 에이전트가 통과나 결격을 확정하면 사용자는 공식 판단처럼 오해할 수 있습니다. 그래서 부족한 항목은 통과가 아니라 보류 또는 확인 필요로 내려야 합니다.

또한 “e나라도움 제출 직전 정리”라는 표현은 로컬 체크리스트를 만들라는 뜻일 수도 있고, 실제 업로드 순서까지 준비하라는 뜻일 수도 있습니다. 이 차이가 정해지지 않으면 에이전트가 검토 도우미로 멈춰야 할지, 외부 서비스로 넘어가려 해야 할지 헷갈릴 수 있습니다.

여기에 한 가지가 더 필요합니다. 검토표 초안이 만들어진 뒤에도 바로 사용자에게 보내면 안 됩니다. 근거가 빠졌는지, 표 형식이 맞는지, 결격 표현이 너무 단정적인지, 민감정보가 들어갔는지를 한 번 더 자동으로 점검해야 합니다.

따라서 지금 상태에서는 완성형 자동화로 바로 만들기보다, 어떤 부분을 더 물어봐야 하는지와 사람이 확인해야 하는 지점을 먼저 정하는 것이 안전합니다.

목표 정리

공고문과 제출서류 폴더의 파일 목록을 대조해 누락 서류, 파일명 불일치, 명시적으로 확인 가능한 자격조건 불충족 항목을 찾고, 사람이 최종 확인할 수 있는 통과/보류/결격 후보 분류표와 e나라도움 제출 전 로컬 체크리스트를 만든다.

업무 흐름

현재 설명만으로 그려 본 업무 순서입니다. 중간에 정보가 부족하면 바로 결과를 만들지 않고 질문으로 돌아가야 합니다.

  1. 1. 요청 목적 확인

    서류 자동 제출이 아니라, 사람이 제출 전에 확인할 수 있는 검토표를 만드는 일인지 확인합니다.

  2. 2. 자료 확인

    공고문, 제출서류 폴더, 신청자 정보, 내부 체크리스트가 어디에 있는지 확인합니다.

  3. 3. 부족한 정보 질문

    자료 위치나 판단 기준이 없으면 바로 표를 만들지 않고 사용자에게 다시 묻습니다.

  4. 4. 점검 순서 정하기

    현재 설명만으로 확인 가능한 항목만 점검 순서에 넣습니다.

  5. 5. 검토표 초안 작성

    확실히 확인할 수 있는 항목만 통과, 보류, 결격 후보로 조심스럽게 나눕니다.

  6. 6. 근거 확인

    누락 서류와 자격조건을 사람이 확인할 수 있는 표로 정리합니다.

  7. 7. 결과물 자동 점검

    근거 누락, 표 형식 오류, 단정적인 표현, 민감정보 포함 여부를 확인합니다.

  8. 8. 사람 확인

    실제 제출, 업로드, 최종 결격 표현은 사람이 확인한 뒤에만 진행합니다.

  9. 9. 결과 전달

    분석 결론, 부족한 정보, 확인 질문을 보고서로 보여줍니다.

중간에 멈출 수 있는 지점

  • e나라도움 제출 범위가 불분명합니다.

    “제출 직전 정리”가 파일 정리인지, 업로드 준비인지, 실제 제출인지 먼저 확인해야 합니다.

  • 자격조건을 확정할 자료가 부족합니다.

    폴더와 공고문만으로 자격조건을 확정할 수 없으면, 결과표를 만들기 전에 질문으로 돌아갑니다.

  • 확인 가능한 항목만 점검해야 합니다.

    문서 본문이나 신청자 정보가 필요한 항목은 확정하지 않고 보류로 둡니다.

  • 검토표가 바로 전달되면 안 됩니다.

    근거가 없거나 표현이 너무 단정적이면 수정으로 돌아가고, 민감정보가 보이면 사람 확인으로 넘깁니다.

  • 최종 판정처럼 보일 수 있습니다.

    사람 검토 전에는 “결격”이 아니라 “결격 후보”처럼 조심스럽게 표현해야 합니다.

부족한 부분

  • 실제 폴더 접근 승인

    제출 폴더에는 개인정보나 민감 문서가 있을 수 있어 승인 없이 읽으면 안 됩니다.

  • 자격조건 판단 자료

    지역, 체납, 동일지원 이력 같은 정보가 없으면 자격조건은 보류해야 합니다.

  • 결격 표현

    사람 검토 전에는 최종 결격이 아니라 결격 후보로 표현해야 합니다.

  • 파일명 허용 규칙

    권장 파일명과 다른 파일을 결격으로 볼지, 보류로 둘지 기준이 필요합니다.

  • 결과물 전달 전 점검 기준

    근거 누락, 형식 오류, 확정 표현, 민감정보 포함 여부를 확인하는 기준이 필요합니다.

  • e나라도움 정리 범위

    로컬 체크리스트인지, 업로드 순서인지, 실제 화면 조작인지 구분해야 합니다.

불분명한 정보와 도구 판정

서비스명, 자료 출처, 사용 도구가 불분명하면 먼저 정의를 요청하고 도구 사용을 보류합니다.

  • 뜻 확인 필요e나라도움이 무엇인지 정의가 필요합니다.

    현재 요청에는 이름만 있고, 공식 사이트인지, 내부 업무명인지, 사람이 화면에서 처리하는 절차인지 알 수 없습니다.

    도구 판단: 뜻을 확인하기 전에는 실제 제출이나 화면 조작을 하지 않고, 로컬 체크리스트 작성까지만 허용합니다.

    확인 질문: e나라도움은 어떤 시스템이고, 이 업무 도우미가 해야 할 일은 파일명 정리, 업로드 준비, 실제 제출 중 무엇인가요?

  • 자료 출처 필요제출서류 폴더를 어떻게 읽을지 정해야 합니다.

    폴더 경로, 샘플 여부, 개인정보 포함 여부, 접근 승인이 아직 정해지지 않았습니다.

    도구 판단: 경로와 승인 범위가 정해진 뒤에만 파일 목록을 읽습니다.

    확인 질문: 실제 제출 폴더를 읽어도 되나요, 아니면 샘플 파일명 목록으로만 설계해야 하나요?

  • 도구 선택 보류HWP/PDF 처리 도구를 아직 확정할 수 없습니다.

    공고문과 제출서류가 HWP인지 PDF인지, 파일명만 볼지 본문까지 볼지 정해지지 않았습니다.

    도구 판단: 본문까지 읽어야 한다고 확인되면 문서 읽기 도구를 검토하고, 파일명 대조만 필요하면 파일 목록 확인으로 제한합니다.

    확인 질문: 공고문과 제출서류는 파일명만 확인하면 되나요, 아니면 HWP/PDF 본문 내용까지 읽어야 하나요?

다음에 정해야 할 일

  • 안전장치제출·업로드·확정 판정 전에 멈추기

    e나라도움 업로드, 외부 제출, 최종 결격 표현이 나오면 업무 도우미가 바로 멈추고 사람 확인을 요청해야 합니다.

  • 반복 절차공고문에서 기준표 뽑기

    공고문에서 필수서류, 제출 형식, 파일명 규칙, 자격조건을 뽑는 순서를 따로 정해 두면 매번 같은 방식으로 검토할 수 있습니다.

  • 표현 규칙검토표에 쓰는 말 정하기

    사람 검토 전에는 `결격`을 최종 판정처럼 쓰지 않고, `결격 후보`처럼 조심스럽게 표현해야 합니다.

  • 자동 점검검토표 전달 전 확인 기준 정하기

    결과표를 보내기 전에 근거 누락, 형식 오류, 확정 표현, 민감정보 포함 여부를 확인하고, 문제가 있으면 수정 또는 사람 확인으로 돌아갑니다.

  • 추가 정보사용자에게 더 받아야 할 정보

    실제 폴더 접근 가능 여부, 자격조건 판단 자료 위치, 내부 체크리스트 존재 여부, 최종 산출물 형식을 더 받아야 합니다.

확인 질문

  • 최종 산출물 범위

    검토표까지만 만들까요, 제출 직전 파일 묶음까지 만들까요?

  • 누락 판단 기준

    공고문 원문 기준으로 볼까요, 내부 체크리스트 기준으로 볼까요?

  • 자격조건 항목

    지역, 업종, 매출, 고용 인원 중 꼭 확인해야 하는 항목은 무엇인가요?

이 보고서는 수업용 예시입니다. 실제 공고문 원문, 제출 파일명 원문, 개인정보, 인증정보를 포함하지 않습니다.
