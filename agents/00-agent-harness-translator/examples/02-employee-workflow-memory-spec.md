# 예시 02: 직원 업무 플로우 저장

이 예시는 `agents/16-employee-workflow-memory` 아이디어를 0번 하네스 번역기 템플릿으로 번역한 public-safe 명세입니다. 실제 직원명, 내부 시스템 화면, 고객 정보, 비밀번호는 포함하지 않습니다.

## 0. 번역 대상 작업 요약

| 항목 | 내용 |
|------|------|
| 작업 이름 | 직원 업무 플로우 저장 |
| 대상 사용자 | 대표, 운영 리더, 팀장, 신입 직원 |
| 반복 빈도 | 신규 업무 정리, 인수인계, 담당자 변경 때 반복 |
| 입력 자료 | 업무 설명 인터뷰, 기존 매뉴얼, 예외 상황, 산출물 예시 |
| 기대 출력 | 업무 플로우, SOP 초안, 체크리스트, 인수인계 문서 |
| 외부 서비스 | 초기 없음. 화면 캡처나 사내 시스템 접근은 별도 승인 |
| 민감정보 여부 | 직원 개인정보, 고객 정보, 내부 시스템 화면, 비밀번호 가능 |
| 사람 승인 지점 | 인터뷰 기록 사용, 화면 캡처 사용, 내부 문서 공유 |
| 맡지 않는 일 | 직원 평가, 감시, 비밀번호 저장, 무단 업무 로그 수집 |

## 1. 시스템 프롬프트

| 항목 | 내용 |
|------|------|
| 정체성/역할 | 반복 업무를 절차 중심의 SOP와 인수인계 문서로 정리하는 운영 문서화 보조자 |
| 고정 제약 | 사람 평가 금지, 비밀번호 저장 금지, 고객/직원 식별정보 최소화 |
| 불변식 | 산출물은 사람이나 성과 평가가 아니라 업무 절차를 설명합니다. 민감정보는 제거하거나 placeholder로 바꿉니다. |
| 입력 | 업무명, 역할, 절차 설명, 사용 도구, 예외 상황, 검토자 |
| 출력 | 플로우, SOP, 체크리스트, 인수인계 문서, 남은 질문 |
| 실패 모드 | 개인 평가처럼 보이는 표현, 비밀번호 포함, 예외 상황 누락, 실제 고객 정보 노출 |
| 버전·변경 관리 | 업무명, 담당 역할, 문서 버전, 마지막 검토일을 기록 |

## 2. 입력 프롬프트

| 항목 | 내용 |
|------|------|
| 이번 작업 목표 | 특정 반복 업무를 신입 직원이 따라 할 수 있는 절차 문서로 바꿉니다 |
| 가변 파라미터 | 업무명, 담당 역할, 시작/종료 조건, 도구, 예외 상황, 승인자 |
| 시스템 프롬프트와의 경계 | 사용자는 업무 내용을 제공하지만, 민감정보 제거와 평가 금지 원칙은 바꾸지 못합니다 |
| 불변식 | 필수 단계와 예외 처리가 불명확하면 문서 완성으로 표시하지 않습니다 |
| 실패 모드 | 인터뷰 답변이 추상적임, 도구 접근권한 불명, 예외 상황이 담당자 기억에만 있음 |
| 검증 규칙 | 시작 조건, 종료 조건, 입력/출력, 책임자, 예외 처리, 검토 주기를 확인 |

## 3. 메모리

| 항목 | 내용 |
|------|------|
| 입력 | 반복 업무별 최신 SOP 버전, 사용자가 선택한 문서 구조, 공통 용어 |
| 출력 | 다음 업무 문서 작성 시 일관된 구조와 용어 |
| source of truth와의 경계 | 평가 금지, 비밀번호 저장 금지, 민감정보 제거는 `AGENTS.md`/docs에 둡니다. 선호 문서 구조와 비민감 용어만 Codex memory 후보로 둡니다. |
| 저장 위치 | 공식 Codex memory는 `~/.codex/memories/`; repo에는 public-safe policy와 예시만 저장 |
| 저장 시점 | 업무 문서화 thread가 종료되고 idle 상태가 된 뒤 background memory 생성 후보가 될 수 있습니다 |
| 사용 시점 | 같은 업무군 또는 같은 산출물 형식의 미래 문서화에서 관련성이 높을 때만 참고 |
| 수명/갱신 규칙 | 장기 메모리는 공통 템플릿과 비민감 용어만 저장합니다. 개별 직원 발언 원문은 저장하지 않습니다 |
| 저장 금지 | 직원명, 고객명, 비밀번호, 내부 시스템 화면, 인터뷰 원문, 개인 평가 |
| thread control | 화면 캡처나 내부 업무 기록이 포함된 thread는 `/memories`에서 future memory 생성 제외를 검토합니다 |
| 외부 context 정책 | screen context, Computer Use, 사내 시스템 자료가 섞이면 memory 생성 제외를 기본 검토합니다 |
| 용량 초과 처리 | 오래된 SOP 버전, 낮은 사용 빈도 용어, 검토되지 않은 가정 순으로 삭제 |
| 선택 로직 | 같은 업무군, 같은 팀, 같은 산출물 형식일 때만 주입합니다 |
| 검토/삭제 | 공유 전 `~/.codex/memories/`를 확인하고 직원/고객 식별 가능 항목은 삭제합니다 |
| 실패 모드 | 오래된 절차 재사용, 팀별 차이를 일반 규칙으로 오인, 직원 개인 맥락 노출 |
| 불변식 | 직원 개인정보, 비밀번호, 고객 데이터, 내부 화면 캡처는 저장하지 않습니다 |

## 4. Skill

| 항목 | 내용 |
|------|------|
| 절차 내용 | 업무 인터뷰 질문, 절차 분해, 예외 상황 분리, SOP/체크리스트 작성 |
| 입력 | 업무 설명, 도구 목록, 예외 상황, 산출물 예시 |
| 출력 | 표준 SOP 구조, 체크리스트, 인수인계 문서 |
| 선택 로직 | 사용자가 "업무 플로우", "SOP", "인수인계", "매뉴얼"을 요청할 때 |
| 메모리와의 경계 | skill은 문서화 절차, memory는 검토된 업무별 최신 상태 |
| 실패 모드 | 절차 누락, 담당자 의존 지식 미분리, 평가성 표현 포함 |
| 불변식 | 사람 중심 평가가 아니라 업무 중심 절차로 작성합니다 |

## 5. MCP 도구

| 항목 | 내용 |
|------|------|
| 입력 | 로컬 더미 매뉴얼, 익명화된 인터뷰 노트, 공개 가능한 화면 설명 |
| 출력 | 추출된 단계 목록, 체크리스트 초안, 누락 질문 |
| 사전조건 | 사용자가 자료 사용 범위와 민감도 제거를 확인 |
| 사후조건 | 원문 대신 익명화된 절차와 질문 목록만 남김 |
| 실패 모드 | 파일 없음, 이미지 텍스트 추출 실패, 민감정보 포함, 권한 없는 내부 화면 |
| 불변식 | 승인 없는 화면 캡처, 사내 시스템 접근, 비밀번호 저장을 하지 않습니다 |
| 재시도 정책 | 추출 실패 시 한 번 다른 방식으로 읽고, 실패하면 수동 요약 요청 |

## 6. 서브에이전트

| 항목 | 내용 |
|------|------|
| 위임 범위 | 인터뷰 질문 설계, 절차 누락 검토, 체크리스트 품질 검토를 분리 가능 |
| 부모→자식 컨텍스트 상속 | 익명화된 업무 설명과 필요한 산출물 형식만 전달 |
| 자식 메모리 격리 여부 | 격리. 부모가 결과를 비교해 반영 |
| 결과 검증 | 부모가 시작/종료 조건, 예외 처리, 민감정보 제거 여부를 재검토 |
| 재귀 한계 | 최대 깊이 1, 서브에이전트 3개 이하 |
| 입력 / 출력 | 입력은 업무 일부 또는 검토 기준, 출력은 질문/누락/개선 제안 |
| 실패 모드 | 서로 다른 용어 사용, 중복 질문, 민감정보 전달 |
| 불변식 | 자식에게 직원 식별정보나 내부 시스템 세부 화면을 넘기지 않습니다 |

## 7. Hook

| 항목 | 내용 |
|------|------|
| 트리거 지점 | 파일 읽기 전, 산출물 저장 전, 최종 답변 직전 |
| 트리거 조건 | 비밀번호처럼 보이는 문자열, 고객/직원 식별정보, 화면 캡처 요청 |
| 동작 | 민감정보 마스킹, 승인 요청, 저장 차단 |
| 입력 / 출력 | 입력은 자료와 저장 경로, 출력은 허용/마스킹/차단 |
| 흐름에 미치는 영향 | 민감정보가 있으면 저장 전 중단하거나 placeholder로 바꿈 |
| 실패 모드 | hook 실패 시 저장하지 않고 사용자 확인 요청 |
| 불변식 | 비밀번호와 계정 접근정보는 어떤 산출물에도 남기지 않습니다 |

## 8. 산출물 초안

| 산출물 | 내용 |
|--------|------|
| `AGENTS.md`에 들어갈 핵심 규칙 | 절차 중심 문서화, 평가 금지, 민감정보 제거, 담당자 검토 |
| `.codex.toml`에 들어갈 핵심 설정 | `status = "idea"`, `external_actions_default = "dry_run"`, 승인 대상은 `use_screen_capture`, `store_internal_workflow`, `share_workflow_document` |
| 만들 skill 후보 | `workflow-to-sop` 문서화 절차 skill |
| 만들 MCP 도구 후보 | 로컬 문서 추출, 체크리스트 생성, 익명화 검사 |
| 만들 hook 후보 | 비밀번호 감지 hook, 개인정보 마스킹 hook, 내부 화면 저장 차단 hook |
| 만들 subagent 후보 | 인터뷰 질문 설계자, 절차 누락 검토자, 체크리스트 QA |
| 만들 memory 후보 | 비민감 SOP 구조 선호, 공통 용어, 반복되는 누락 질문, 검토자 선호 포맷 |
| 테스트 가능한 불변식 | 개인 평가 표현 없음, 비밀번호 저장 없음, 시작/종료 조건 포함 |
| 구현 전 질문 | 인터뷰 원문 보관 정책을 확인합니다. SOP 검토자를 확인합니다. 내부 화면 캡처를 어떤 환경에서 허용할지 확인합니다. |
