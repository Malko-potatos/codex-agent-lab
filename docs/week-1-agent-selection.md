# Week 1 Agent Selection

작성일: 2026-05-26

이 문서는 네 개의 아이디어 대화에서 첫 주차에 만들 에이전트를 하나씩 선발한 기록입니다. 기준은 이 저장소의 원칙을 따릅니다.

이 문서의 목적은 “무엇이 멋져 보이는가”를 고르는 것이 아닙니다. 수강생이 첫 주에 안전하게 만들어 볼 수 있는 작은 업무 도우미를 고르는 것입니다.

## 선발 기준

- 로컬 초안 생성부터 시작할 수 있어야 합니다.
- 실제 고객/회사/직원/인증정보를 repo에 넣지 않아도 검증 가능해야 합니다.
- 외부 전송, 업로드, 게시, 로그인 세션 사용 없이 첫 주차 산출물을 만들 수 있어야 합니다.
- `AGENTS.md`, `.codex.toml`, memory contract, sample input/output으로 작게 닫을 수 있어야 합니다.
- 사용자가 일상 업무에서 반복적으로 체감할 수 있는 가치가 있어야 합니다.
- Codex runtime 수정이나 복잡한 멀티에이전트 오케스트레이션이 필요하지 않아야 합니다.

## 대화 1: 회사 자료 기반 지원사업 사업계획서

### 판정

첫 주차 후보로 선발합니다. 단, 실제 목표인 `.hwp` 완성본 자동작성까지 가지 않고, **지원사업 요구사항 대응표 + 사업계획서 markdown 초안 + 누락자료 체크리스트**까지만 만듭니다.

### 선발 이유

- 사용자가 느끼는 반복 업무 가치가 큽니다.
- 회사 자료가 많아 token budget, memory, retrieval-like 자료 선별 전략을 실험하기 좋습니다.
- 기존 `01-business-plan-hwp` 에이전트의 목적과 직접 맞습니다.
- 첫 주차에는 실제 회사 문서를 쓰지 않고도 더미 자료로 구조를 검증할 수 있습니다.

### 첫 주차 불변식

- 실제 회사 자료, 사업자등록증, 증명서, 기존 사업계획서 원문은 repo에 저장하지 않습니다.
- `.hwp` 직접 생성은 첫 주차 범위에서 제외합니다.
- 금액, 실적, 인증, 인력, 시장수치에는 출처 또는 placeholder가 있어야 합니다.
- 마스터 사업계획서 업데이트는 자동 반영하지 않고 `update candidates` 목록으로만 제안합니다.
- 지원사업 제출, 파일 업로드, 외부 서비스 변환은 하지 않습니다.

### 첫 주차 산출물

- public-safe 더미 회사 프로필
- public-safe 더미 지원사업 공고/평가기준
- 자료 인벤토리 schema
- 요구사항 대응표 template
- 사업계획서 markdown 초안 template
- 누락 제출서류 체크리스트
- memory contract: 회사의 반복 문체/목차 선호는 memory 후보, 실제 회사 자료와 공고 원문은 저장 금지

## 대화 2 최종 선발: 만족도조사 설문폼 생성

### 선발 이유

- 기존 프로그램별 설문 패턴을 더미 데이터로 쉽게 재현할 수 있습니다.
- Google Forms API 없이도 수정 가능한 JSON/markdown schema를 만들 수 있습니다.
- 개인정보 수집 최소화, 편향 문항 점검, human review를 명확히 설계할 수 있습니다.
- 첫 주차에 샘플 입력/출력, 검증 규칙, form schema까지 작게 완성 가능합니다.

### 첫 주차 불변식

- 실제 응답자 목록, 이메일, 전화번호, 설문 응답 데이터는 repo에 저장하지 않습니다.
- Google Forms 생성/배포는 하지 않습니다.
- 개인정보 문항은 기본 제외하고, 필요 시 목적과 보관 정책을 명시합니다.
- 유도 질문, 과도한 긍정 편향, 중복 문항을 검토합니다.
- 출력은 사람이 수정 가능한 초안이어야 합니다.

### 첫 주차 산출물

- public-safe 프로그램 정보 샘플
- 설문 문항 template
- form schema JSON 예시
- 문항 편향/개인정보 체크리스트
- 프로그램 유형별 섹션 구성 예시
- memory contract: 선호 척도/문체/섹션 순서는 memory 후보, 응답자 정보와 실제 응답은 저장 금지

## 대화 3 최종 선발: 사내 문서 자동생성 시스템

### 선발 이유

- 반복 문서 작업 절감 효과가 큽니다.
- 템플릿과 변수 schema만으로 첫 주차 검증이 가능합니다.
- 실제 급여, 세금, 계약, 고객 자료를 쓰지 않고도 더미 문서로 구조를 확인할 수 있습니다.
- 기존 `09-document-automation-system` 에이전트의 목적과 직접 맞습니다.

### 첫 주차 불변식

- 실제 급여명세서, 세금계산서, 계약서, 고객 문서, 계좌정보는 repo에 저장하지 않습니다.
- 첫 주차 대상 문서는 public-safe 공문과 더미 견적서로 제한합니다.
- `.docx`, `.pdf`, `.hwp` 생성은 첫 주차 범위에서 제외하고 markdown preview로 검증합니다.
- 금액, 세금, 법률 문구는 확정하지 않고 placeholder 또는 검토 필요로 표시합니다.
- 발송, 전자서명, 세금계산서 제출, 외부 업로드는 하지 않습니다.

### 첫 주차 산출물

- 문서 유형별 변수 schema
- public-safe 공문 template
- public-safe 견적서 template
- 더미 입력 JSON 예시
- markdown preview 출력 예시
- 민감정보 마스킹 체크리스트
- memory contract: 반복 문서 형식과 문체 선호는 memory 후보, 실제 문서 원문과 민감 변수는 저장 금지

## 대화 4 최종 선발: 행사 홈페이지 제작

### 선발 이유

- 실제 개인정보나 인증 없이 더미 행사 brief로 즉시 검증할 수 있습니다.
- 결과물이 눈에 보이기 때문에 일상 업무 사용자가 Codex agent의 가치를 빠르게 체감할 수 있습니다.
- 첫 주차에는 정적 구조와 문구 초안만 만들고, 신청/조회/배포는 명확히 제외할 수 있습니다.
- 기존 `14-event-website-builder` 에이전트의 목적과 직접 맞습니다.

### 첫 주차 불변식

- 실제 참가자 정보, 연락처, 결제 정보, 비공개 행사 정보는 repo에 저장하지 않습니다.
- Wix, 아임웹, 서버, 도메인, 배포 연동은 하지 않습니다.
- 참가신청과 조회 기능은 구현하지 않고 placeholder로만 둡니다.
- 레퍼런스 URL과 키비주얼은 원문 복제하지 않고 구조/톤 참고 수준으로만 다룹니다.
- 이미지 권리와 초상권이 확인되지 않은 자산은 사용하지 않습니다.

### 첫 주차 산출물

- public-safe 행사 brief 샘플
- 행사 페이지 섹션 schema
- 한/영 카피 초안 template
- 정적 홈페이지 구조 markdown 또는 HTML 초안
- 배포 전 체크리스트
- 신청/조회 기능 요구사항 placeholder
- memory contract: 선호 섹션 순서와 톤앤매너는 memory 후보, 실제 참가자 정보와 비공개 행사 정보는 저장 금지
