# Agent Idea Review Checklist

각 에이전트 아이디어를 실제 구현하기 전에 아래 항목을 검토합니다.

## 1. 반복성과 가치

- 이 업무가 월 1회 이상 반복되는가?
- 사람이 하는 데 걸리는 시간이 명확한가?
- 자동화하면 초안 작성, 검토, 제출 중 어느 단계가 줄어드는가?
- 실패했을 때 사람이 쉽게 복구할 수 있는가?

## 2. 입력 데이터

- 필요한 입력이 문서, 표, 이미지, 메신저, 웹페이지 중 무엇인가?
- 입력 데이터에 개인정보, 계약정보, 급여정보, 세금정보가 포함되는가?
- 샘플 데이터로 기능을 검증할 수 있는가?
- 공개 저장소에 올려도 되는 더미 데이터가 있는가?

## 3. 산출물

- 최종 산출물이 초안, 검토표, 자동 제출물 중 무엇인가?
- 파일 형식은 `.hwp`, `.hwpx`, `.docx`, `.xlsx`, `.pdf`, 웹폼, 게시글 중 무엇인가?
- 사람이 최종 검토해야 하는 지점은 어디인가?

## 4. 외부 연동

- 네이버웍스, 네이버카페, 이메일, Google Forms, CMS, Instagram 등 외부 서비스가 필요한가?
- 읽기 전용으로 충분한가, 쓰기/전송 권한이 필요한가?
- OAuth, 서비스 계정, 쿠키, 수동 로그인 중 어떤 인증 방식이 필요한가?
- 자동 실행 전에 사용자 승인을 받을 수 있는가?

## 5. 법적/정책 리스크

- 크롤링 대상 서비스의 약관을 위반할 가능성이 있는가?
- 개인 연락처나 기자 DB를 자동 발송에 사용하는가?
- 공공 지원사업 문서에 허위/과장 내용이 생성될 위험이 있는가?
- AI가 만든 내용을 사람이 검토하고 책임질 프로세스가 있는가?

## 6. 구현 우선순위

- 1단계: 로컬 초안 생성
- 2단계: 샘플 데이터 기반 검증
- 3단계: 내부 사용자 검토
- 4단계: 외부 연동 dry-run
- 5단계: 승인 기반 제한적 운영
