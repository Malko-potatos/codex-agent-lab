# 쉬운 용어 사전

이 문서는 비전공 대학생이나 직장인이 Codex 에이전트 만들기를 이해할 수 있도록 어려운 용어를 쉬운 말로 풀어 설명합니다. 기준은 파인만 학습법입니다. 즉, 어떤 용어든 “친구에게 설명할 수 있을 만큼 쉬운 말”로 바꿔 봅니다.

설명의 편의를 위해 반복해서 나오는 용어는 이 문서에 명시해 둡니다. 다른 문서에서 같은 단어가 나오면 이 용어집의 뜻을 기준으로 읽으면 됩니다.

## 이 저장소를 한 문장으로 설명하면

이 저장소는 Codex를 기반으로 반복 문서 업무와 운영 업무를 자동화하기 위한 에이전트를 만드는 스터디 자료입니다.

여기서 에이전트는 대단한 인공지능 제품이라기보다, “반복되는 업무를 맡길 수 있도록 역할과 규칙을 정리한 작은 업무 도우미”라고 생각하면 됩니다.

## 핵심 용어

| 용어 | 쉬운 설명 |
|------|-----------|
| Codex | 문서, 코드, 파일, 설정을 함께 다루며 일을 도와주는 AI 작업 파트너입니다. |
| 에이전트 | 특정 반복 업무를 맡기기 위해 만든 작은 역할 담당자입니다. 예를 들어 “만족도조사 초안 담당자”, “사업계획서 초안 담당자”가 에이전트입니다. |
| 하네스 | 에이전트가 안전하게 일하도록 잡아주는 작업대입니다. 무엇을 읽고, 무엇을 만들고, 어디서 멈출지 정합니다. |
| 프롬프트 | AI에게 주는 업무 지시서입니다. “무엇을 해 주세요”라고 적는 요청문입니다. |
| 시스템 프롬프트 | 에이전트가 항상 지켜야 하는 기본 규칙입니다. 회사의 업무 원칙이나 안전 규칙에 가깝습니다. |
| 입력 프롬프트 | 이번 한 번의 작업에서만 필요한 요청입니다. 예를 들어 “이번 행사 만족도조사를 만들어 주세요” 같은 문장입니다. |
| `AGENTS.md` | 해당 폴더에서 Codex가 따라야 하는 업무 매뉴얼입니다. 사람으로 치면 “이 부서에서 일하는 법”입니다. |
| `.codex.toml` | 에이전트의 설정표입니다. 이름, 목적, 입력, 출력, 필요한 도구, 승인 규칙을 정리합니다. |
| 메모리 | 다음에 같은 설명을 반복하지 않기 위해 남겨두는 기억입니다. 단, 비밀번호나 고객 자료처럼 민감한 내용은 기억하면 안 됩니다. |
| skill | 자주 반복되는 절차를 카드처럼 정리한 것입니다. 예를 들어 “제안서 요구사항 대응표 만드는 법”이 skill이 될 수 있습니다. |
| MCP | Codex가 외부 도구와 연결될 때 쓰는 통로입니다. 쉽게 말하면 AI와 다른 서비스 사이의 안전한 콘센트입니다. |
| hook | 위험한 행동 전에 멈춰 세우는 검문소입니다. 예를 들어 이메일 발송 전 “정말 보낼까요?”라고 확인하게 만드는 장치입니다. |
| subagent | 일을 나눠 맡기는 보조 담당자입니다. 예를 들어 한 명은 자료를 찾고, 한 명은 표를 검토하게 나누는 방식입니다. |
| dry-run | 실제 발송, 업로드, 제출을 하지 않고 리허설만 해보는 방식입니다. |
| 초안 우선 | 처음부터 완성본을 만들지 않고 사람이 검토할 수 있는 초안을 먼저 만든다는 원칙입니다. |
| 사람 검토 필수 | AI가 만든 결과를 사람이 확인한 뒤에만 제출, 발송, 게시한다는 원칙입니다. |
| public-safe | 공개 저장소에 올려도 위험하지 않은 상태입니다. 실제 고객 자료, 개인정보, 비밀번호가 없어야 합니다. |
| mock data | 실제 자료 대신 연습용으로 만든 가짜 데이터입니다. 수업에서는 mock data로 먼저 연습합니다. |
| schema | 입력이나 출력의 빈칸 양식입니다. 예를 들어 견적서에는 `항목`, `수량`, `단가`, `합계` 같은 칸이 필요합니다. |

## 헷갈리기 쉬운 구분

### 메모리와 `AGENTS.md`

메모리는 “다음에도 참고하면 좋은 개인적인 기억”입니다.

`AGENTS.md`는 “반드시 지켜야 하는 업무 규칙”입니다.

예를 들어 “이 수업에서는 실제 고객 자료를 저장하지 않습니다”는 `AGENTS.md`나 문서에 있어야 합니다. 반면 “사용자는 표 형태 요약을 선호합니다”는 메모리 후보가 될 수 있습니다.

### skill과 프롬프트

프롬프트는 이번 작업을 부탁하는 말입니다.

skill은 여러 번 반복해서 쓸 수 있는 절차입니다.

예를 들어 “이번 행사 설문지를 만들어 주세요”는 프롬프트입니다. “좋은 만족도조사 문항을 만드는 순서”는 skill입니다.

### MCP와 hook

MCP는 외부 도구와 연결하는 통로입니다.

hook은 위험한 순간에 멈추는 검문소입니다.

예를 들어 Google Forms를 만드는 도구와 연결하는 것은 MCP입니다. 실제 설문 링크를 배포하기 전에 승인을 요구하는 것은 hook입니다.

## 스터디에서 기억할 원칙

1. 처음부터 자동 제출을 만들지 않습니다. 먼저 초안을 만듭니다.
2. 실제 자료 대신 mock data로 연습합니다.
3. 민감정보는 저장하지 않습니다.
4. 외부 발송, 업로드, 게시 전에는 사람이 승인합니다.
5. 어려운 용어는 쉬운 예시로 바꿔 설명합니다.
